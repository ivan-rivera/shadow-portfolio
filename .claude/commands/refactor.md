# Refactor Changes

## Overview
Refactor the codebase to comply with best practices and achieve concise, readable code.

## Steps

1. **Minimise lines of code**
   - Refactor the codebase to minimise lines of code
   - Avoid duplication as much as possible
2. **Simplify logic**
   - Simplify the logic of the codebase
   - Avoid complex logic as much as possible
3. **Improve readability**
   - Improve the readability of the codebase
   - Ensure classes and functions are named appropriately
   - Ensure docstrings are present, up to date, and easy to follow
4. **Improve performance**
   - Improve the performance of the codebase
